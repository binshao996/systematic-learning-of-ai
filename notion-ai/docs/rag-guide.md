# RAG 技术入门指南

## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与大语言模型生成相结合的技术架构。它通过从外部知识库中检索相关文档片段，将其作为上下文注入到 LLM 的提示词中，从而让模型能够基于外部知识回答问题。

RAG 的核心思想是：**不要求 LLM 记住所有知识，而是在需要时动态检索相关信息**。

## RAG 的工作流程

### 1. 文档摄入阶段

文档摄入是将原始文档转换为可检索格式的过程，包含三个关键步骤：

- **文档解析**：将 PDF、Word、Markdown 等格式的文档解析为纯文本。
- **文本分块**：将长文本切分为适当大小的片段。常见的分块策略有固定大小分块、语义分块和递归分块。
- **向量化**：使用 Embedding 模型将每个文本块转换为高维向量。这些向量捕捉了文本的语义信息。
- **索引存储**：将向量存储到向量数据库（如 Qdrant、Milvus、Pinecone）中，以便后续快速检索。

### 2. 查询阶段

当用户提出问题时，RAG 系统执行以下步骤：

- **查询向量化**：将用户的问题通过相同的 Embedding 模型转换为向量。
- **相似度检索**：在向量数据库中搜索与查询向量最相似的 Top-K 个文档块。通常使用余弦相似度作为度量标准。
- **上下文组装**：将检索到的文档块与系统提示词、用户问题一起组装成完整的 Prompt。
- **生成回答**：将组装好的 Prompt 发送给 LLM，生成最终回答。

## 分块策略详解

分块是 RAG 系统中最重要的环节之一，好的分块策略能显著提升检索质量：

### 固定大小分块
将文本按照固定 Token 数量切分。优点是简单高效，缺点是可能切断完整语义。适合日志、代码等结构化文本。

### 语义分块
根据文档的自然边界（段落、章节标题）进行切分。保持语义完整性，检索结果更准确。适合文章、书籍等结构化文档。

### 递归分块
尝试多种分隔符（双换行 → 单换行 → 句号 → 空格）进行递归切分，直到块大小合适。这是 LangChain 的默认策略，通用性最强。

## Embedding 模型选择

Embedding 模型的质量直接影响检索效果：

| 模型 | 维度 | 特点 |
|------|------|------|
| BGE-large-zh | 1024 | 中文效果优秀，开源免费 |
| text-embedding-3-small | 1536 | OpenAI 出品，多语言支持 |
| Jina-embeddings-v3 | 1024 | 多语言，支持可变维度 |
| M3E-large | 768 | 中文专用，轻量高效 |

## 向量数据库对比

- **Qdrant**：Rust 编写，性能优秀，支持过滤和批量操作，适合中小规模应用。
- **Milvus**：分布式向量数据库，支持十亿级向量检索，适合企业级应用。
- **Pinecone**：全托管云服务，零运维，按量付费。
- **Chroma**：轻量级，Python 原生，适合原型开发。

## 混合检索策略

单纯依赖向量相似度可能遗漏关键词完全匹配的结果。混合检索结合了：

1. **语义检索**：基于 Embedding 向量相似度
2. **关键词检索**：基于 BM25 算法的传统文本匹配

最终得分通过加权组合或 RRF（Reciprocal Rank Fusion）算法融合。

## RAG 的常见挑战

1. **检索不相关**：检索到的文档块与问题无关，导致 LLM 生成错误回答。
2. **上下文窗口溢出**：检索到的文档块过多，超出 LLM 的上下文限制。
3. **信息丢失**：答案分散在多个文档块中，单个块无法覆盖完整信息。
4. **幻觉问题**：检索到的信息不足时，LLM 可能编造事实。

## 总结

RAG 是目前最实用的 LLM 应用架构之一。它通过将检索和生成相结合，既保留了 LLM 的语言能力，又解决了知识截止和幻觉问题。构建一个高质量的 RAG 系统需要在文档解析、分块策略、Embedding 模型和检索策略等多个环节进行优化。
