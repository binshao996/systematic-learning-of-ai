# 文档解析管道 — 结构化学习笔记

## 1. PDF 解析

### 库概览

| 库 | 许可证 | 基于 | 打包体积 | 解析速度 | 表格支持 | 图片支持 |
|---|---|---|---|---|---|---|
| `unpdf` | MIT | PDF.js | ~1.5 MB | 快速 | 不支持（纯文本） | 不支持（仅元数据） |
| `pdf-parse` | MIT | pdf.js（旧版） | ~1.2 MB | 中等 | 不支持 | 不支持 |

两个库底层都封装了 Mozilla 的 `pdf.js`。`unpdf` 是更现代、仍在维护的封装库，支持 TypeScript，API 也更简洁。

### 使用 `unpdf`

```typescript
import { readFileSync } from "node:fs";
import { getDocument, extractText } from "unpdf";

async function parsePdfWithUnpdf(filePath: string) {
  const fileBuffer = readFileSync(filePath);
  const pdf = await getDocument(fileBuffer);

  // 按页提取文本
  const { text, pages } = await extractText(pdf);

  // `text` — 所有文本通过换行符拼接
  // `pages` — { pageNumber, content } 数组，附带元数据

  const pageCount = pages.length;
  const allText = pages.map((p) => p.content).join("\n\n");

  return {
    text: allText,
    pageCount,
    pages: pages.map((p) => ({ pageNumber: p.pageNumber, content: p.content })),
  };
}
```

### 使用 `pdf-parse`

```typescript
import { readFileSync } from "node:fs";
import pdf from "pdf-parse";

async function parsePdfWithPdfParse(filePath: string) {
  const fileBuffer = readFileSync(filePath);
  const data = await pdf(fileBuffer);

  // `data.text` — 提取的全部文本
  // `data.numpages` — 页数
  // `data.info` — PDF 元数据（标题、作者等）

  return {
    text: data.text,
    pageCount: data.numpages,
    metadata: data.info,
  };
}
```

### 表格和图片 — 局限性

PDF 是一种**呈现格式**，而非结构化数据格式。表格以定位文本块的方式渲染，没有行列感知。`unpdf` 和 `pdf-parse` 都：

- 将表格内容返回为**扁平文本**（无网格结构）
- 无法提取矢量图形或内嵌图片
- 丢失多栏布局（文本会跨越栏目从左到右读取）

从 PDF 中提取表格需要专用工具：

```typescript
// 表格需要专门的后处理或使用类似 `pdf-table-extractor` 的库
// 启发式示例：检测重复的空白模式
function detectTableLines(lines: string[]): string[][] {
  return lines
    .filter((line) => line.split(/\s{2,}/).length > 2)
    .map((line) => line.split(/\s{2,}/));
}
```

同样的局限性适用于图片 — 只能获取元数据（尺寸、编码），无法获取像素数据。

### 页码追踪

`unpdf` 原生支持页码：

```typescript
const { pages } = await extractText(pdf);
for (const page of pages) {
  console.log(`第 ${page.pageNumber} 页: ${page.content.slice(0, 80)}...`);
}
```

`pdf-parse` 不暴露每页内容。可以通过单独渲染每一页作为替代方案：

```typescript
async function parseByPage(filePath: string) {
  const fileBuffer = readFileSync(filePath);
  const pdf = await getDocument(fileBuffer);

  const pageTexts: { pageNumber: number; content: string }[] = [];
  for (let i = 1; i <= pdf.totalPages; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    pageTexts.push({
      pageNumber: i,
      content: content.items.map((item: any) => item.str).join(" "),
    });
  }
  return pageTexts;
}
```

## 2. Word (.docx) 解析

### 库：`mammoth`

`mammoth` 将 `.docx` 文件转换为 HTML 或纯文本。它是**有损**的 — 复杂布局可能无法保留 — 但它比任何 PDF 解析器都能更好地捕获语义结构（标题、列表、表格）。

```typescript
import { readFileSync } from "node:fs";
import mammoth from "mammoth";

async function parseDocx(filePath: string) {
  const buffer = readFileSync(filePath);

  // 方案 A：提取纯文本
  const { value: text } = await mammoth.extractRawText({ buffer });

  // 方案 B：提取为 HTML（保留结构）
  const { value: html } = await mammoth.convertToHtml({ buffer });

  return { text, html };
}
```

### 提取带层级的标题

`mammoth` 将 Word 标题样式（`Heading 1`、`Heading 2` 等）映射为 HTML `<h1>`、`<h2>` 等。通过自定义转换可以捕获层级结构：

```typescript
interface DocxSection {
  heading: string;
  headingLevel: number;
  content: string;
}

async function parseDocxWithHeadings(filePath: string): Promise<DocxSection[]> {
  const buffer = readFileSync(filePath);
  const { value: html } = await mammoth.convertToHtml({ buffer });

  // 解析 HTML 以提取按标题锚定的节
  const sections: DocxSection[] = [];
  const headingRegex = /<(h[1-6])>([^<]+)<\/\1>/g;
  const parts = html.split(headingRegex);
  // parts: [before, tag, text, between, ...]

  // 更简单的方法：按标题标签分割
  const sectionBlocks = html.split(/(<(h[1-6])>.*?<\/\2>)/s);
  let currentHeading = "";
  let currentLevel = 0;
  let currentContent = "";

  for (const block of sectionBlocks) {
    const match = block.match(/<(h([1-6]))>(.*?)<\/\1>/);
    if (match) {
      if (currentHeading) {
        sections.push({
          heading: currentHeading,
          headingLevel: currentLevel,
          content: currentContent.trim(),
        });
      }
      currentHeading = match[3];
      currentLevel = parseInt(match[2], 10);
      currentContent = "";
    } else if (block.trim()) {
      // 去除 HTML 标签获得纯文本内容
      currentContent += block.replace(/<[^>]*>/g, "").trim() + "\n";
    }
  }

  // 推入最后一个节
  if (currentHeading) {
    sections.push({
      heading: currentHeading,
      headingLevel: currentLevel,
      content: currentContent.trim(),
    });
  }

  return sections;
}
```

### 表格处理

`mammoth` 将 Word 表格转换为 HTML `<table>` 元素。可以通过后处理步骤提取：

```typescript
async function parseDocxTables(filePath: string) {
  const buffer = readFileSync(filePath);
  const { value: html } = await mammoth.convertToHtml({ buffer });

  // 从 HTML 中提取表格
  const tableRegex = /<table>[\s\S]*?<\/table>/g;
  const tables = html.match(tableRegex) ?? [];

  return tables.map((tableHtml, i) => {
    const rows = tableHtml.match(/<tr>[\s\S]*?<\/tr>/g) ?? [];
    const cells = rows.map((row) => {
      const cols = row.match(/<td>[\s\S]*?<\/td>/g) ?? [];
      return cols.map((c) => c.replace(/<\/?td>/g, "").trim());
    });
    return { index: i, rows, cells };
  });
}
```

### 图片处理

`mammoth` 可以通过提供 `convertImage` 选项来提取内嵌图片：

```typescript
interface ExtractedImage {
  altText: string;
  contentType: string;
  data: Buffer;
}

async function parseDocxWithImages(filePath: string) {
  const buffer = readFileSync(filePath);
  const images: ExtractedImage[] = [];
  let imageIndex = 0;

  const { value: html } = await mammoth.convertToHtml({
    buffer,
    convertImage: mammoth.images.imgElement(async (image) => {
      const imageBuffer = await image.read();
      images.push({
        altText: image.altText ?? `image-${imageIndex}`,
        contentType: image.contentType,
        data: imageBuffer,
      });
      imageIndex++;
      return { src: `image-${imageIndex - 1}` };
    }),
  });

  return { html, images };
}
```

## 3. Markdown 解析

### 方案 A：纯文本 + 正则表达式（轻量级）

对于只需要标题层级的简单场景，基于正则表达式的方法无需任何依赖：

```typescript
interface MarkdownSection {
  heading: string;
  headingLevel: number;
  content: string;
}

function parseMarkdownSimple(markdown: string): MarkdownSection[] {
  const sections: MarkdownSection[] = [];
  const lines = markdown.split("\n");
  let currentHeading = "root";
  let currentLevel = 0;
  let currentContent: string[] = [];

  for (const line of lines) {
    const headingMatch = line.match(/^(#{1,6})\s+(.+)$/);
    if (headingMatch) {
      // 保存上一节
      if (currentHeading) {
        sections.push({
          heading: currentHeading,
          headingLevel: currentLevel,
          content: currentContent.join("\n").trim(),
        });
      }
      currentLevel = headingMatch[1].length;
      currentHeading = headingMatch[2];
      currentContent = [];
    } else {
      currentContent.push(line);
    }
  }

  // 推入最后一节
  sections.push({
    heading: currentHeading,
    headingLevel: currentLevel,
    content: currentContent.join("\n").trim(),
  });

  return sections;
}
```

### 方案 B：`remark` / `unified`（基于 AST）

如需完全控制（代码块、链接、图片、表格），可使用 `unified` 生态：

```bash
bun add unified remark-parse remark-stringify remark-gfm mdast-util-from-markdown
```

```typescript
import { unified } from "unified";
import remarkParse from "remark-parse";
import remarkStringify from "remark-stringify";
import { visit } from "unist-util-visit";
import type { Root, Heading, Code, Link, Paragraph } from "mdast";

interface ParsedMarkdown {
  sections: SectionInfo[];
  codeBlocks: CodeBlockInfo[];
  links: LinkInfo[];
}

interface SectionInfo {
  heading: string;
  headingLevel: number;
  content: string;
}

interface CodeBlockInfo {
  language: string | null;
  code: string;
}

interface LinkInfo {
  text: string;
  url: string;
}

function parseMarkdownAst(markdown: string): ParsedMarkdown {
  const processor = unified().use(remarkParse);
  const tree = processor.parse(markdown);

  // 提取标题结构
  const sections: SectionInfo[] = [];
  const headings: Heading[] = [];

  visit(tree, "heading", (node: Heading) => {
    headings.push(node);
    const headingText = (node.children as Paragraph[])
      .map((child) => child.children?.[0]?.value ?? "")
      .join("");
    sections.push({
      heading: headingText,
      headingLevel: node.depth,
      content: "", // 需要遍历标题之间的子节点
    });
  });

  // 提取代码块
  const codeBlocks: CodeBlockInfo[] = [];
  visit(tree, "code", (node: Code) => {
    codeBlocks.push({
      language: node.lang ?? null,
      code: node.value,
    });
  });

  // 提取链接
  const links: LinkInfo[] = [];
  visit(tree, "link", (node: Link) => {
    const linkText = (node.children as Paragraph[])
      .map((child) => child.children?.[0]?.value ?? node.url)
      .join("");
    links.push({ text: linkText, url: node.url });
  });

  return { sections, codeBlocks, links };
}
```

### 提取标题之间的节内容（AST）

AST 遍历不会自动将内容归属到对应标题下。一种更彻底的方式是直接使用树结构：

```typescript
function splitIntoSectionContent(markdown: string): SectionInfo[] {
  const processor = unified().use(remarkParse);
  const tree = processor.parse(markdown) as Root;
  const sections: SectionInfo[] = [];
  let currentSection: SectionInfo | null = null;
  let contentBuffer = "";

  for (const node of tree.children) {
    if (node.type === "heading") {
      if (currentSection) {
        currentSection.content = contentBuffer.trim();
        sections.push(currentSection);
      }
      const headingText = (node as Heading).children
        .map((child: any) => child.value ?? "")
        .join("");
      currentSection = {
        heading: headingText,
        headingLevel: (node as Heading).depth,
        content: "",
      };
      contentBuffer = "";
    } else if (currentSection) {
      // 将节点序列化回 markdown 文本作为内容
      const contentNode = node as any;
      if (contentNode.value) {
        contentBuffer += contentNode.value + "\n";
      } else if (contentNode.children) {
        contentBuffer +=
          contentNode.children
            .map((c: any) => c.value ?? "")
            .join("") + "\n";
      }
    }
  }

  if (currentSection) {
    currentSection.content = contentBuffer.trim();
    sections.push(currentSection);
  }

  return sections;
}
```

### 对比：正则 vs. AST

| 方面 | 正则 | remark/unified AST |
|---|---|---|
| 依赖 | 无 | `unified` + 插件（~200 KB） |
| 解析速度 | 非常快（~100万字符/秒） | 较慢（~10万字符/秒） |
| 边界情况准确性 | 遗漏嵌套/怪异的语法 | 完全遵循规范 |
| 代码块提取 | 需要自定义逻辑 | 内置通过 `visit` |
| 链接元数据 | 用正则匹配 `[text](url)` | 内置 `link` 节点类型 |
| 标题层级 | 简单数 `#` | 深度 + 嵌套感知 |

**推荐**：快速标题提取和简单节分割使用正则。需要可靠的代码块检测、链接提取或 GFM 表格解析时使用 `remark`。

## 4. 统一解析器设计

### 解析器接口

每种解析器实现返回的统一输出类型：

```typescript
interface ParsedDocument {
  text: string;                        // 全部纯文本
  sections: Section[];                 // 按标题锚定的节
  metadata: {
    sourceFile: string;
    fileType: "pdf" | "docx" | "md" | "txt";
    pageCount: number | null;          // 非分页格式为 null
    headings: string[];                // 所有标题的扁平列表
  };
}

interface Section {
  heading: string;
  headingLevel: number;
  content: string;
  pageNumber: number | null;           // 无页码信息时为 null
  hasTable: boolean;
  hasCode: boolean;
}
```

### 文件路由器

路由器检查文件扩展名并委托给相应的解析器：

```typescript
type FileType = "pdf" | "docx" | "md" | "txt";

type ParserFn = (filePath: string) => Promise<ParsedDocument>;

function detectFileType(filePath: string): FileType {
  const ext = filePath.split(".").pop()?.toLowerCase() ?? "";
  const extMap: Record<string, FileType> = {
    pdf: "pdf",
    docx: "docx",
    md: "md",
    markdown: "md",
    txt: "txt",
    text: "txt",
  };
  if (ext in extMap) return extMap[ext];
  return "txt"; // 兜底
}

async function parseDocument(filePath: string): Promise<ParsedDocument> {
  const fileType = detectFileType(filePath);
  const parsers: Record<FileType, ParserFn> = {
    pdf: parsePdf,
    docx: parseDocx,
    md: parseMarkdown,
    txt: parseTextFile,
  };
  return parsers[fileType](filePath);
}

// 纯文本文件解析器示例（兜底方案）
async function parseTextFile(filePath: string): Promise<ParsedDocument> {
  const content = readFileSync(filePath, "utf-8");
  return {
    text: content,
    sections: [{ heading: "", headingLevel: 0, content, pageNumber: null, hasTable: false, hasCode: false }],
    metadata: {
      sourceFile: filePath,
      fileType: "txt",
      pageCount: null,
      headings: [],
    },
  };
}
```

### 按文件类型的节检测

每种解析器将其原生结构映射到 `Section` 接口：

| 文件类型 | 原生结构 | 节数据来源 | 页码来源 |
|---|---|---|---|
| PDF | 按页排列的文本块 | 启发式：检测全大写/短行作为标题 | 通过 `unpdf` 内置支持 |
| DOCX | Word 样式（标题 1/2/3） | `mammoth` HTML 标题标签 | 不可用 |
| Markdown | `#` / `##` / `###` | 正则或 remark AST | 不可用 |
| 纯文本 | 扁平内容 | 单个节 | 不可用 |

### 设计考量

1. **优雅降级**：如果解析器无法提取节（例如没有文本层的扫描 PDF），则回退到纯文本文件处理器，使用单个节。

2. **流式处理大型文档**：对于超过 10 MB 的文件，考虑使用流式处理：
   ```typescript
   async function* parseDocumentStream(filePath: string): AsyncGenerator<Section> {
     // 逐个产出节，而不是缓冲整个文档
   }
   ```

3. **缓存解析输出**：将 `ParsedDocument` 序列化到缓存文件（JSON 或二进制），避免重复解析：
   ```typescript
   const cachePath = filePath + ".parsed.json";
   if (existsSync(cachePath)) {
     return JSON.parse(readFileSync(cachePath, "utf-8"));
   }
   ```

4. **解析器注册表模式**：通过注册表使文件类型到解析器的映射可扩展：
   ```typescript
   class ParserRegistry {
     private parsers = new Map<FileType, ParserFn>();
     register(fileType: FileType, parser: ParserFn) {
       this.parsers.set(fileType, parser);
     }
     get(fileType: FileType): ParserFn {
       return this.parsers.get(fileType) ?? this.parsers.get("txt")!;
     }
   }
   ```
